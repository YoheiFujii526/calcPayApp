# イベント精算アプリ 設計書

## 1. システム概要

### 1.1 システム名

**研究室向けイベント精算管理アプリ**

### 1.2 目的

研究室内で行われる飲み会、旅行、イベントなどにおいて、誰がいくら支出し、誰がいくら返済したかを管理し、最終的な精算額を明確にすることを目的とする。

### 1.3 背景

研究室のイベント会計では、以下のような課題が発生しやすい。

* 誰が立て替えたのか把握しづらい
* 誰が返済済みか分かりにくい
* 支出対象が全員とは限らない
* 最終的に誰が誰へいくら払えばよいか計算が面倒

これらの課題を解決するため、本システムではイベント単位で参加者・支出・返済情報を管理し、未精算額や最終的な支払い先まで可視化する。

## 10. 精算ロジック概要

### 10.1 基本方針

各支出について、対象者間で均等割りし、各参加者の本来負担額を算出する。
その後、各参加者について以下を求める。

* 実際に支払った金額
* 本来負担すべき金額
* 受け取った返済額
* 支払った返済額
* 最終差額

### 10.2 計算式

```text
差額 = 実際の支払額 - 本来負担額 + 受け取った返済額 - 支払った返済額
```

* 差額が正: その人は受け取る側
* 差額が負: その人は支払う側

### 10.3 支払い提案例

受け取る側と支払う側を金額順に突き合わせて、誰が誰へいくら払えばよいかを生成する。

例:

* A: +5000
* B: -3000
* C: -2000

結果:

* B → A に 3000円
* C → A に 2000円

### 10.4 実装責務

このロジックは `SettlementService` に実装する。
